## Deviations Documented

- DEV-01 — AbortSignal not threaded into spawnSync (Severity: Medium; pre-existing, amplified by the raised 10-min default). Original spec assumed the timeout threads into spawnSync and 'no further change is needed there'; actual exec closure checks opts.signal?.aborted before/after spawnSync but does not pass the signal into spawnSync. Reason: constraints forbade touching the control-flow engine and did not request abort changes. Impact: a mid-compile abort on a 10+ min budget runs to full timeout before the post-spawn check fires. Recommended follow-up: pass signal: opts.signal into spawnSync options (verify Node>=16 engines first). Maps to CR-01.
- DEV-02 — Two dedupe paths (Severity: Low; DRY/maintainability, no behavior change). A module-level dedupePreservingOrder() handles the opts.testPackages branch while parseTestPackages() re-implements the same trim/filter/dedupe-order-preserving logic inline for the env branch. Verified behaviorally equivalent by tests. Recommended optional follow-up: have parseTestPackages call dedupePreservingOrder. Maps to CR-02.
- DEV-03 — Generic non-rust scoping intentionally deferred (Severity: Info). Spec allowed generic build-system detection 'where possible' with Cargo as the minimum. Actual implementation guards scoping on language==='rust' only; go/python/node/mixed stacks produce byte-identical argv regardless of SUPER_DEV_BUILD_TEST_PACKAGES. Reason: spec's surgical-fix recommendation; the env-var + opts.testPackages plumbing is generic enough to extend to npm/pnpm/go later without an API change. Impact: none for backward compatibility (non-rust repos behave exactly as before per AC-06); Rust was the only slow-compiling workspace class that triggered the false FAILs that killed specs 54/55.
