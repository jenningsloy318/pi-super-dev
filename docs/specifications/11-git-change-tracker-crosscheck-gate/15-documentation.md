## Deviations Documented

- Deviation 1 — Phase end split into probeEnd + commitEnd (src/tracking.ts). Spec specified a single end(unit,id,claimed?) called once after the phase attempt loop; implementation uses probeEnd per-attempt (computes freshest cross-check, stashes last-record, does NOT persist) + commitEnd once after the loop (persists exactly the final record). Reason: the gate must read a cross-check on every attempt to drive the retry loop, but a single end would either lose the per-attempt signal or produce duplicate jsonl phase-end lines. Surfaced by code review CR-MED. Impact: none to false-green contract or jsonl record shape (stages still use single end unchanged).
- Deviation 2 — computeCrossCheck normalizes paths before matching via new pure normalizeTrackerPath() in src/tracking.ts. Spec specified exact string equality for the claimedNotChanged set difference; implementation normalizes both sides (backslash→POSIX, collapse //, strip leading ./ and leading /, strip trailing /, case-preserving) while keeping original strings in output arrays. Reason: without normalization an LLM claim like ./src/x.ts, src//x.ts, or src\x.ts would be spuriously flagged claimedNotChanged against git's repo-relative src/x.ts — a false-red changeGate FAIL, the opposite of the conservative posture SCENARIO-006 mandates. Surfaced by code review CR-HIGH. Impact: none to clean-path cases (33 tracking.test.ts cross-check cases unchanged).
- Non-deviations confirmed against code and documented in the spec: stage bracketing via the ctx.events subscription seam (minimal touch, no nodes.ts/pipeline.ts internals); activeTracker singleton mirroring activeRun with set in pipeline.ts at state.setup finalization + clear in extension.ts execute() finally; computeChangeGate co-located in build-runner.ts with the spec's exact signature/semantics; structured-change contract in buildImplementPrompt/buildFixPrompt with legacy flat array tolerated by parseStructuredChanges; spec-10 claimed.filesCreated→requireFiles union with no circular double-count; zero new runtime deps via gitSpawn(argv) reusing spawnSync + resolveTimeoutMs.
