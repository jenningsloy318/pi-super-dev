## Phase 1: Touched-crate detection helper — ✅ COMPLETE (commit 89bdc763)

domain: build-runner. Added `detectTouchedCargoPackages(cwd, baseRef?)`: spawns `git -C <cwd> diff --merge-base <baseRef> --name-only` (baseRef = arg > `SUPER_DEV_GATE_BASE_REF` > `"main"`), maps `crates/<pkg>/…` lines to `<pkg>` via `/(?:^|\/)crates\/([^/]+)\//`, dedupes via `dedupePreservingOrder`, returns `[]` on any git error / empty diff / non-crate-only diff, never throws (single try/catch). Pure-input unit tests in `tests/build-runner-touched-crates.test.ts` stubbing `spawnSync` verify path→pkg mapping, dedupe, order, git-failure→`[]`, non-crate→`[]`, and base-ref override. Covers AC-01 → SCENARIO-001/002/003/020/022/023. depends_on: none. parallelizable_with: Phase 2.

## Phase 4: In-scope failure classification on BuildGateResult — ✅ COMPLETE (commit 88553c07)

domain: build-runner. Extended `BuildGateResult` with additive `outOfScopeErrors: string[]` and `inScopePass: boolean`; added pure `classifyOutOfScopeErrors(errors, scopedSet)` extracting `crates/<pkg>/` (from `--> path` markers) and `-p <pkg>` markers — the command-label `-p` is excluded by scanning `-p` only on the post-` FAILED (` flagRegion — marking an error OUT-OF-SCOPE iff it references ≥1 crate AND all referenced crates are NOT in scope (conservative in-scope on ambiguity/no-marker); computed `inScopePass = !pass && errors.length>0 && outOfScopeErrors.length===errors.length` (true when pass; stays false when no scoping active). Never throws. Wired into runBuildGate after error collection. Classifier unit tests in `tests/build-runner-inscope-classification.test.ts`. Covers AC-04 → SCENARIO-009/010/011/021/024/028. depends_on: Phase 3.
## Phase 5: In-scope verdict in implementation retry loop — ✅ COMPLETE (commit f9b93ffb)

Stage tests in `src/stages/implementation.test.ts` assert green-via-inscopepass proceeds (no early abort, commit runs), genuine in-scope failures still terminate, and pass-path unchanged. Covers AC-05 → SCENARIO-012/013/014/025/027. depends_on: Phase 4.
