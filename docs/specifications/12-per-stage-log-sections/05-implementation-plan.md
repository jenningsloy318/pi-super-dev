## Phase 2: groupByStage pure partitioner — ✅ COMPLETE

AC-02 (SCENARIO-005..009). Add a new dependency-free module `src/render/stage-grouping.ts` exporting `groupByStage(transcript, statusOf?): StageGroup[]` where `StageGroup = { stageId; stageLabel; status?; lines: Array<{kind,text}> }`. Partitions tagged entries preserving FIRST-APPEARANCE order, strips stage tags from emitted `lines` (callers consume only `{kind,text}`), coalesces every untagged OR string-shaped entry into ONE sentinel fallback group (`stageId:"setup"`, `stageLabel:"pre-stage"`), returns `[]` for empty input, and resolves each group's status via an INJECTED `statusOf?: (stageId) => string | undefined` lookup (undefined when absent — keeps the helper pure/unit-testable, no dashboard import). Independently testable in isolation with synthetic tagged/untagged transcripts. Domain: render-stage-grouping. Depends on the widened `TranscriptLine` shape from Phase 1.
## Phase 3: Streaming per-stage sections in flush() — ✅ COMPLETE

AC-03 (SCENARIO-010..013). Rebuild `flush()` in `src/render/live-stream.ts` to render a STACK of per-stage sections via `groupByStage`. Add named constants `RUNNING_TAIL_LINES = 15`, `COMPLETED_TAIL_LINES = 3`, and `TOTAL_SECTION_CAP`. Each section = a status-themed header line (running→accent+bold+animated braille glyph, ok→success, failed→error, skipped→warning/dim) followed by that stage's lines themed per-kind via `themeLine` and indented two spaces; a blank line separates sections. The running stage shows ≤`RUNNING_TAIL_LINES` recent lines; completed stages render COMPACT (header + ≤`COMPLETED_TAIL_LINES` tail, or header-only). Per-stage `trim` notices (kind `trim`) appear INSIDE their own section. TUI header carries a leading `▌` bar in the status color. Mode gate unchanged: `mode==="tui" && theme` enables theming; every other mode emits RAW TEXT (plain `▶ Stage N` headers + indented logs) with ZERO ANSI bytes (AC-08 byte-clean preserved). All theme access method-style via existing `fg`/`bold` wrappers. PARALLELIZABLE WITH Phase 4 (both depend only on Phases 1+2 and touch disjoint files). Independently testable by asserting section structure, tail caps, and the no-ANSI non-TUI regression. Domain: render-live-stream.
