Impact: coverage equivalent, no behaviour change.
- D-2: requireTests originally specified to match each declared test name against the WHOLE raw test-list stdout blob → implementation matches PER-LINE. Reason: a substring hit in a path/dir-header/comment line could falsely satisfy the contract. Impact: correctness hardening; verified non-regressing against all SCENARIO-005/006 fixtures.
- D-3: runDeliverableCheck signature gained opts.skipTests (spec had only {signal,timeoutMs}); implementation stage passes skipTests:!buildGreen. Reason: on a build-red attempt the test list is stale/meaningless and spawning it wastes the cached subprocess. Impact: performance optimization; on build-green attempts requireTests runs exactly as specified (deferred Low finding).
- D-4: implementation.ts GREEN condition is exactly (gate.pass||gate.inScopePass)&&deliverableCheck.pass as specified, PLUS an explicit resetDeliverableCheckCache() call at the run boundary (module-level testListCache Map reset). Reason: without the reset the cached cargo/vitest test list from a prior run would leak (stale test names) across runs. Impact: correctness for multi-run processes; one extra import; behaviour-identical within a single run.
