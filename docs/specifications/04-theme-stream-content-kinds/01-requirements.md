## Executive Summary

Today the super-dev live stream and result render flatten every content kind (commands, thinking, process logs, phase markers, errors, corrective notices) into one undifferentiated text block, whereas pi-native separates them with distinct foreground colors AND tool-bubble backgrounds. This is a pure TypeScript change to src/extension.ts + src/render/dashboard.ts + a new pure module src/render/stream-theme.ts that classifies each sink line into one of 10 LineKinds and themes it via pi's real Theme tokens (toolTitle/thinkingText/accent/warning/error/success/muted + toolPendingBg/toolSuccessBg), with graceful degradation to raw text in print/json/headless modes. The bridge already routes all stream content through exactly three ProgressSink methods (phase/log/text), so classification lives entirely at the sink layer with no upstream emit-site changes.

## Non-Functional Requirements

- Performance: classifyLine/themeLine are pure and called once per stream line; themeLine must short-circuit to the raw string when theme is undefined so non-TUI flush pays only a single identity-return per line (no ANSI allocation). The streaming flush already throttles at FLUSH_MS; the new per-line styling must not introduce per-flush allocations beyond the join that already exists.
- Backward/byte-compatibility: print/json/RPC/headless and the saved disk log MUST remain ANSI-free â€” theme-absent is identity, and the on-disk write uses raw line.text (no kind, no codes) so existing log-grep tooling and snapshots are unaffected.
- Dependencies: zero new runtime dependencies; reuse pi-tui Text/Container/Markdown and the real pi Theme tokens only.
- Correctness: classifyLine ordering is order-sensitive (command-done before command; error/warning/success keyword scans after the marker checks) and must left-trim whitespace because sink.log indents stored lines with two spaces (extension.ts:229).
- Strict TypeScript: tsconfig is strict and the module is ESM (`type:module`); the extended DashboardTheme optional members must not force changes on existing importers of the interface.
